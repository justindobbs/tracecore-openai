What you are looking at

This repo adapts OpenAI Agents SDK patterns into app-like examples with a local UX and an explicit validation story. The goal is not just to show prompts working once, but to explain the path from demo behavior to repeatable verification.

  • Chat Assistant is a lightweight single-agent surface for onboarding, product guidance, and basic question handling.
  • Support Triage demonstrates multi-agent routing logic for billing, technical, and account-style issues.
  • Deterministic verification mode makes local validation cheap and repeatable before live-model testing.

TraceCore workflow

  1. Run the app locally with uvicorn tracecore_openai.main:app --reload.
  2. Exercise the routes in this UI to understand the intended behavior.
  3. Run tracecore-openai-verify to execute deterministic scenarios.
  4. Inspect deliverables/verification/latest.json to confirm the expected outputs were observed.
  5. Only then move into live OpenAI-backed usage and richer app expansion.

Verification and runtime notes

  • Set TRACECORE_OPENAI_FAKE_RUNNER=1 when you want deterministic local runs without API spend.
  • Run tracecore-openai-verify to execute the fixed scenario suite and produce a JSON report.
  • Live mode uses the OpenAI Agents SDK directly, so results can vary more than deterministic verification outputs.
  • The current starter repo proves the loop first: build -> try -> verify -> expand.

What TraceCore testing is doing here

The starter verification harness is intentionally simple: it runs known prompts through deterministic app mode and confirms that expected outputs or routing hints show up. That gives you a stable baseline before you add more realistic tasks, validators, or benchmark-style run artifacts.
