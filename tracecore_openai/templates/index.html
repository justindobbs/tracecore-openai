TraceCore Journey

1. Run the example

Use the Chat Assistant or Support Triage tabs to show that the agent works as an application surface and responds to real prompts.

  • The first signal is simple: does the agent respond correctly in the UI?
  • This proves usefulness, but not reliability.

2. Verify the agent

Run TraceCore verification to see whether the agent meets deterministic expectations and reliability standards across known scenarios.

  • Verification results tell you whether behavior is dependable, not just plausible.
  • If it fails, the output gives you evidence about what to fix next.
From demo to certified agent

3. Remediate when verification fails

If the verification report shows weak routing, inconsistent phrasing, or missing required behaviors, you refine the agent instructions, tools, routing, or tests and run verification again.

4. Bundle verify for release confidence

Once the agent passes the relevant checks, you move to bundle verification so the release artifact carries evidence that the agent meets the expected TraceCore standard and can be treated as a certified bundle.

Verification

Step 2 through step 5 of the workflow live here: run deterministic scenarios, inspect reliability evidence, improve the agent if needed, and finish with bundle verification for a certifiable artifact.

  • Set TRACECORE_OPENAI_FAKE_RUNNER=1 for deterministic local runs without API spend
  • Run tracecore verify --latest as the native TraceCore reliability check
  • Use tracecore inspect --run <run_id> to review a stored run artifact
  • Use tracecore diff <run_a> <run_b> or tracecore baseline --compare ... after changes
  • Use tracecore runs list and tracecore runs summary so the workflow feels like pytest for agents
  • Once behavior is stable, move to bundle sealing and verification

uvicorn tracecore_openai.main:app --reload
tracecore verify --latest
tracecore inspect --run <run_id>
tracecore diff <run_a> <run_b>
tracecore baseline --agent <agent> --task <task> --compare <run_a> <run_b>
tracecore runs list --limit 5
tracecore runs summary --limit 5
python -m pytest
python -m ruff check tracecore_openai tests

tracecore bundle seal --latest
tracecore bundle verify <bundle_dir>
tracecore bundle sign <bundle_dir>
What TraceCore Testing Proves

TraceCore testing answers the question a user asks after seeing an agent work once: is this agent actually reliable? The verification harness runs known scenarios through deterministic app mode and checks whether the required outputs or routing decisions appear. A passing result gives evidence of reliability. A failing result gives a concrete target for remediation. After that loop is green, bundle verification gives you the release-time proof for a certified agent artifact.
