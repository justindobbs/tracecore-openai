Workspace context

What this dashboard is showing

App surfaces

This repo adapts OpenAI Agents SDK patterns into app-like examples with a local UX and an explicit validation story.

  • Chat Assistant is a lightweight single-agent surface for onboarding, product guidance, and basic question handling.
  • Support Triage demonstrates multi-agent routing logic for billing, technical, and account-style issues.

Operator workflow

  1. Run the app locally with uvicorn tracecore_openai.main:app --reload.
  2. Exercise the routes in this dashboard to understand intended behavior.
  3. Run tracecore-openai-verify to execute deterministic scenarios.
  4. Inspect deliverables/verification/latest.json before moving deeper into live usage.

Planning forward

Verification meaning and next upgrades

What TraceCore testing is doing here

The starter verification harness is intentionally simple: it runs known prompts through deterministic app mode and confirms that expected outputs or routing hints show up. That gives you a stable baseline before you add more realistic tasks, validators, or benchmark-style run artifacts.

Where this can go next

Once this landing page and workflow feel right, the next upgrades are richer result history, inline verification status cards, scenario browsing, and direct links to generated artifacts so the user can move from demo interaction to evidence without leaving the UI.
